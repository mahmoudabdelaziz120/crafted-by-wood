
## الخطة النهائية

### 1) تفعيل Lovable Cloud + المصادقة
- تفعيل Lovable Cloud (قاعدة بيانات + Auth آمنة).
- إنشاء حساب admin بالإيميل: `ahmed@alkashlan.com` (لأن Supabase يشترط إيميل صالح — سأشرح للمستخدم أنه يسجل دخول بهذا الإيميل بدل `AHMED@ALKASHLAN` لأنها ليست صيغة إيميل سليمة) وكلمة المرور: `123456789AHMED`.
- جدول `user_roles` منفصل (admin/user) لتفادي ثغرة تصعيد الصلاحيات، مع دالة `has_role()` بصلاحيات SECURITY DEFINER.
- صفحة `/admin/login` عامة + كل صفحات الداشبورد تحت `/_authenticated/admin/*` بحيث تُحمى تلقائياً + فحص إضافي على role=admin.

### 2) الجداول في قاعدة البيانات
- `site_settings` (صف واحد): عنوان الـHero، الوصف، نصوص أزرار CTA، رقم واتساب، إحصائيات (عدد المشاريع، سنوات الخبرة، الالتزام بالمواعيد).
- `services` (7 صفوف مبدئية): slug، الاسم بالعربي/الإنجليزي، الوصف، السعر التقريبي، المميزات، الأيقونة، صورة الـHero.
- `gallery_images`: صور المعرض لكل خدمة (service_id, image_url, title, description, order_index).
- `contact_messages`: رسائل الفورم (الاسم، الهاتف، الخدمة، الرسالة، is_read, created_at).
- RLS: قراءة عامة لـ site_settings/services/gallery، كتابة للـ admin فقط. رسائل: إدخال عام (anon) + قراءة/تعديل للـ admin فقط.
- GRANT statements لكل جدول.

### 3) الداشبورد `/admin/*` — مستوحاة من الملف اللي رفعته لكن أنظف
- **`/admin/login`**: صفحة تسجيل دخول بتصميم glassmorphism ذهبي.
- **`/admin`** (Overview): كروت إحصائية (عدد الرسائل الجديدة، عدد الخدمات، آخر تحديث)، آخر 5 رسائل، اختصارات سريعة.
- **`/admin/hero`**: تحرير عنوان ونصوص الصفحة الرئيسية + رقم واتساب.
- **`/admin/stats`**: تعديل الإحصائيات الظاهرة تحت الـHero.
- **`/admin/services`**: قائمة بالـ7 خدمات، عند الضغط تفتح صفحة تحرير (الاسم، الوصف، صورة الـHero، المميزات).
- **`/admin/gallery`**: رفع/حذف/إعادة ترتيب صور المعرض لكل خدمة (Supabase Storage).
- **`/admin/messages`**: عرض الرسائل، تحديد كمقروء، رد سريع على واتساب برابط جاهز، حذف.
- **`/admin/settings`**: تغيير كلمة المرور، تسجيل خروج.

### 4) ربط الصفحة الرئيسية بالداشبورد
- تحويل `services.ts` من ثابت إلى fetching من قاعدة البيانات (بـ `useSuspenseQuery` + server fn public).
- الـ`index.tsx` يقرأ `site_settings` عبر server function عامة (بمفتاح publishable + سياسة SELECT للـanon).
- الصور تبقى محلية كـ fallback، لكن أي صورة مرفوعة من الداشبورد تعلو عليها.

### 5) فورم التواصل + إشعار واتساب
- الفورم في `index.tsx` يستدعي server fn يخزّن الرسالة في `contact_messages`.
- بعد التخزين: يفتح `wa.me/{رقم_واتساب}?text={نص_الرسالة}` في تبويب جديد → المالك يستقبل الرسالة على واتساب فوراً وأيضاً محفوظة في الداشبورد.
- Validation بـ Zod (اسم، هاتف مصري، رسالة max 1000).

### 6) استبدال Lovable AI؟
لا، مش محتاج AI للداشبورد.
